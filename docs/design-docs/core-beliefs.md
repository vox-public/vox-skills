# Core Beliefs

1. `AGENTS.md`는 index이지 encyclopedia가 아니다.
2. installable skill topology는 코드보다 문서가 먼저 결정한다.
3. description은 trigger를 위한 것이고, execution semantics는 `SKILL.md` body가 소유한다.
4. installable skill의 logical namespace는 flat하다.
5. router skill은 얇아야 하며 domain skill을 다시 삼켜서는 안 된다.
6. skill semantics는 skill-local에 두고, harness runner는 repo-level에 둔다.
7. generated docs는 current state를 설명하고, constitutional docs는 허용 구조를 설명한다.

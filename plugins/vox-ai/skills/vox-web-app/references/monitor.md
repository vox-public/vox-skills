# 모니터링 (Monitor) 영역

vox.ai 웹 앱의 **모니터링** 그룹은 통화 결과를 분석하고, 이상을 감지하는 곳이다. 분석, 통화 기록, 알림 3가지 페이지.

---

## 1. 분석 (`/dashboard/{orgId}/analytics`)

통화 성과를 차트로 시각화. 6가지 차트와 다양한 필터를 지원한다.

### 1.1 차트 종류
1. **통화 수** — 기간별 총 통화 건수
2. **통화 시간** — 누적 또는 평균 통화 시간
3. **비용** — 누적 크레딧 소비
4. **평균 통화 시간** — 건당 평균
5. **평균 비용** — 건당 평균
6. **종료 사유 파이 차트** — 에이전트 종료 vs 사용자 종료 vs 기타

### 1.2 기간 그룹화
- **일 / 주 / 월** 단위 선택
- 날짜 범위 선택 (캘린더)

### 1.3 필터 사이드바
- 에이전트
- 배치(캠페인)
- 통화 유형 (웹/전화)
- 유저 감정 (긍정/중립/부정)
- 종료 사유

필터는 실시간 반영되며, 여러 필터를 조합할 수 있다.

---

## 2. 통화 기록 (`/dashboard/{orgId}/history`)

개별 통화 단위로 상세 데이터를 조회하는 페이지. 대시보드에서 가장 많이 쓰는 페이지 중 하나.

### 2.1 테이블 기본 칼럼
- **통화 일시**
- **에이전트** / **버전**
- **통화 시간** (분:초)
- **통화 타입** (웹/전화)
- **비용**
- **연결 종료 사유**: "에이전트 종료"(AI가 end_call) / "사용자 종료"(상대방이 끊음)
- **유저 감정**: 긍정/중립/부정 (자동 분류)

### 2.2 동적 추출 칼럼
- 에이전트에 **통화정보 추출 필드**를 설정했다면, 각 필드가 자동으로 칼럼으로 추가됨
- 예: "신용카드여부", "방문가능여부" 같은 커스텀 필드가 값(true/false/문자열)로 표시

### 2.3 상단 도구

| 도구 | 설명 |
|------|------|
| 캘린더 아이콘 | 날짜 범위 선택 |
| 필터 버튼 | 필터 패널 열기 |
| 컬럼 설정 (격자) | 표시할 칼럼 토글 |
| 새로고침 | 데이터 갱신 |
| 다운로드 ↓ | 현재 필터 + 칼럼 상태로 엑셀 내보내기 (비동기 export 작업) |

### 2.4 필터 패널 항목
- Call ID
- 에이전트
- 발신번호 / 표시번호 / 수신번호
- 연결 종료 사유
- 통화 유형 (웹/전화)
- 유저 감정
- 날짜 범위

### 2.5 통화 상세 시트 (우측 슬라이드 패널)

row 클릭 → 우측에서 슬라이드되는 상세 시트가 열림:

- **메타데이터**: 시간, 지속시간, 상태, 종료 사유
- **에이전트 정보**: 사용된 버전, 태그, 플로우
- **동적 변수 데이터**: 발신 당시 주입된 커스텀 변수 값
- **녹취 재생**: 음성 파일 타임라인 (재생/일시정지/구간 스크러빙)
- **대화 전사**: 메시지 순서, 화자 표시
- **통화 로그**: 상세 단계별 로그 (플로우 전이, 도구 호출 등)

> 녹취 재생과 타임라인 스크러빙은 UI에서만 가능. MCP 도구로 `get_call`은 메타데이터 정도만.

### 2.6 딥링크
- `?callId={call_uuid}` → 해당 통화의 상세 시트를 바로 오픈한 상태로 페이지 열기

### 2.7 엑셀 다운로드
- 우측 상단 **다운로드 ↓**
- 현재 필터/날짜/칼럼 상태로 비동기 Export 작업 생성
- 완료되면 알림 + 파일 다운로드
- 엑셀 파일에는 동적 추출 칼럼 포함

> **권장 분석 흐름**: 발신 기록(deploy.md 4)에서 캠페인 요약 확인 → 통화 기록에서 해당 캠페인으로 필터 → 추출 칼럼으로 핵심 지표 확인 → 엑셀 다운로드 후 피벗/차트 분석.

## 3. 알림 (`/dashboard/{orgId}/alerts`)

통화 지표가 특정 조건을 만족하면 이메일/웹훅/Slack으로 알림을 보내는 규칙 관리.

### 3.1 목록
- 등록된 알림 규칙 카드
- 각 카드: 이름, 조건 요약, 활성 상태 토글, 최근 트리거 시각

### 3.2 규칙 생성
- "+ 새 규칙" 또는 **딥링크 `?create`** → 생성 폼 자동 오픈

#### 폼 항목
- **이름**
- **메트릭**: 예) 평균 통화 시간, 종료 사유=사용자 종료 비율, 총 비용 등
- **연산자**: `>=`, `<=`, `==`, `>`, `<`
- **임계값**
- **시간 윈도우**: 최근 1시간/24시간/7일
- **활성 시간대** (월~일 HH:MM)
- **알림 채널**:
  - 이메일 (수신자 리스트)
  - 웹훅 URL + 서명 키
  - Slack (워크스페이스 연동)

### 3.3 규칙 상세 (`/dashboard/{orgId}/alerts/{ruleId}`)
- 규칙 편집
- **테스트 웹훅** 버튼 → 현재 설정으로 테스트 알림 수동 발송 (웹훅 정상 작동 확인)
- 트리거 이력

## 평가(Evals) — 모니터링의 확장

`/dashboard/{orgId}/evals` (사이드바에 별도 그룹)

- **단위 테스트**: 특정 입력에 대한 에이전트 응답을 검증 (deterministic)
- **시나리오 테스트**: 여러 턴의 가상 대화로 에이전트를 검증
- **테스트 결과**: 실행 이력 조회

딥링크:
- `?create=unit` → 단위 테스트 생성 페이지로
- `?create=scenario` → 시나리오 테스트 생성 페이지로

> 세부 평가 전략/프롬프트 진단 방법은 `vox-agents` 스킬 참조.

# 배포 (Deploy) 영역

vox.ai 웹 앱의 **배포** 그룹은 실제 전화 발신/수신을 설정하는 곳이다. 전화번호 관리, 단건 발신, 대량 발신, 발신 기록 4가지 페이지를 포함한다.

---

## 1. 번호 관리 (`/dashboard/{orgId}/numbers`)

보유 번호 목록, 번호 구매, SIP 트렁크 연동, 인바운드 에이전트 설정.

### 1.1 구조

좌우 분할 레이아웃 (DashboardResizableLayout):
- **왼쪽**: 번호 목록 (메모/전화번호 표시)
- **오른쪽**: 선택한 번호의 상세 (인바운드 에이전트, 아웃바운드 에이전트, SIP 정보 등)

### 1.2 번호 구매

- 좌측 상단 "+" 드롭다운 → "새 번호 구매" 또는 "SIP 트렁크 연동"
- 또는 **딥링크 `?new=1`**: 구매 다이얼로그 자동 오픈
  - 예: `https://www.tryvox.co/dashboard/{orgId}/numbers?new=1`

#### 구매 게이트 (Purchase Gate)
구매 전 아래 조건을 모두 만족해야 한다:
1. **인증(verification)**: 신원/조직 인증 완료
2. **결제 수단(billing)**: 등록된 결제 수단 필요
3. **번호 한도(limit)**: 조직 등급(verificationGrade)별 최대 번호 수 이하

조건 미충족 시:
- `limit` 차단 → **번호 한도 다이얼로그**가 열리며 현재 개수/최대 개수 안내
- `verification`/`billing_key` 차단 → **통합 인증 다이얼로그**(identity → organization → billing)
- 모두 통과 시 → **새 번호 구매 다이얼로그**

### 1.3 SIP 트렁크 연동

- 드롭다운 "+ > SIP 트렁크 연동" → 연동 다이얼로그
- 외부 SIP 트렁크(자사 PBX, 타사 통신사) 정보 입력 → `custom` 타입 번호로 등록
- 연동 후 MCP 도구(`list_telephone_numbers`)에서도 조회됨

### 1.4 인바운드 에이전트 설정

- 번호 선택 → 오른쪽 상세에서 **인바운드 에이전트** 필드 수정
- 수신 전화가 걸려올 때 이 에이전트가 응답한다

### 1.5 아웃바운드 테스트 콜 (헤더 버튼)

- 번호 상세 헤더의 **테스트 콜 버튼** → 지정 에이전트로 즉시 아웃바운드 콜 실행

> 번호 삭제/취소/복구 등은 번호 카드의 컨텍스트 메뉴에서 수행.

## 2. 단건 발신 (`/dashboard/{orgId}/outbound/single`)

특정 수신자 1명에게 테스트 통화를 거는 페이지. 동적 변수 입력 → 즉시 발신 → 로그 확인 흐름.

### 2.1 폼 항목

- **에이전트 선택** (드롭다운)
- **발신 번호(call_from)** (보유 번호 중)
- **수신 번호(call_to)** 입력
- **동적 변수**: 에이전트 프롬프트의 `{{변수명}}`에 해당하는 필드가 자동 표시됨 → 값 입력
- **발신 실행** 버튼 → 백엔드에 create_call 호출

### 2.2 실행 후

- 발신 시도 상태 실시간 표시
- 완료 시 `call_id` 반환 → 통화 기록(history) 페이지 링크 제공
- 페이지 내 SDK/API 호출 스니펫(JavaScript/Python/cURL)도 함께 표시됨 (복사해서 백엔드에 붙여넣기 가능)

### 2.3 사전 선택 (딥링크)
- `?numberId={id}` → 발신 번호 사전 선택 상태로 페이지 열기

## 3. 대량 발신 (`/dashboard/{orgId}/outbound/batch`)

다수의 대상에게 자동으로 전화를 거는 페이지. **프로젝트(스프레드시트) + 캠페인(발신 실행 건)** 구조.

### 3.1 프로젝트 생성

대량 발신은 **프로젝트** 단위로 관리한다. 프로젝트 = 발신 대상 리스트가 담긴 스프레드시트. 하나의 프로젝트에서 여러 캠페인(1차/2차/재발신 등)을 만든다.

#### 방법 1: 직접 생성
1. "+ 새 프로젝트 생성" 클릭
2. 빈 스프레드시트 열림
3. 칼럼 추가 (+) 후 발신 대상 데이터 직접 입력
4. 필수 칼럼: **전화번호** (수신자 번호)

#### 방법 2: 파일 업로드
1. 엑셀(.xlsx) 또는 CSV 파일 준비
2. 새 프로젝트 생성 시 파일 업로드 → 데이터 자동 반영
3. **칼럼명이 프롬프트의 `{{변수명}}`과 일치**하면 발신 시 자동으로 변수에 주입됨

- 프로젝트는 이름으로 검색 가능
- 여러 시트(탭) 추가 가능

> CSV 업로드, 스프레드시트 편집은 UI에서만 가능. MCP 도구로 대체 불가.

### 3.2 발신 설정 및 실행

1. 스프레드시트에서 발신할 **row를 체크박스로 선택** (전체 선택 가능)
2. 오른쪽 상단 **"발신"** 버튼 → **전화 발신 설정** 모달

#### 모달 항목
| 항목 | 설명 |
|------|------|
| **캠페인 이름** | 이 발신 건을 식별할 이름 (예: "3월 매장방문 필터링 1차") |
| **에이전트 선택** | 드롭다운. 선택 시 **추출** 항목이 자동 표시 |
| **발신번호 / 표시번호** | 발신 070번호 + 수신자에게 표시될 번호 |
| **동시 통화 수** | -/+ 조절 (기본 5). 클수록 빨리 끝나지만 동시 비용 증가 |
| **발송 시점** | "즉시 발송" 또는 "예약 발송"(날짜/시간 지정) |
| **발신 허용 시간대** | 기본 월~일 08:00~20:00. 밖에서는 일시 중단 후 자동 재개 |
| **완료 알림 수신자** | 이메일 태그로 완료 알림 수신자 등록 |

확인 클릭 → 발신 시작.

> **사전 확인**: 에이전트에 통화정보 추출 필드를 미리 설정하면 캠페인 결과 분석이 쉬워진다. (build.md 1.3 참고)

## 4. 발신 기록 (`/dashboard/{orgId}/outbound/batch-history`)

캠페인별 발신 결과를 한눈에 본다.

### 4.1 테이블 칼럼
- **이름**: 캠페인 이름
- **상태**: 완료 / 진행중
- **예약 | 발신 | 수신**: 각 단계별 건수 (예: `1 > 1 > 0`)
- **비용**: 사용 크레딧
- **발신번호 / 표시번호**
- **시작일시**

### 4.2 상단 도구
- **날짜 필터** (캘린더 아이콘)
- **캠페인 검색**
- **다운로드 ↓**: 캠페인 요약 CSV

### 4.3 캠페인 상세
- row 클릭 → 해당 캠페인의 개별 통화 목록
- 더 상세한 통화 단위 조회/필터링/녹취는 **통화 기록**(monitor.md) 페이지에서

### 4.4 사전 선택 (딥링크)
- `?batchCampaignIds={id1},{id2}` → 특정 캠페인만 필터된 상태로 열기

## 번호 없음 상태 (Zero-state)

조직이 보유 번호가 없는 경우:
- 번호 목록이 빈 상태 (Skeleton 로드 후)
- "번호 구매" 버튼으로 구매 다이얼로그 오픈
- 단건/대량 발신 페이지에서도 발신 번호 선택 불가 → 먼저 번호 구매 안내

> LLM 에이전트가 사용자를 돕는 시나리오: `list_telephone_numbers` 결과가 비어있으면 `https://www.tryvox.co/dashboard/{orgId}/numbers?new=1`로 안내해서 즉시 구매 다이얼로그를 열게 한다.

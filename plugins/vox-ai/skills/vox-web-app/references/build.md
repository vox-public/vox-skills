# 구축 (Build) 영역

vox.ai 웹 앱의 **구축** 그룹은 에이전트의 재료를 만드는 곳이다. 에이전트, 보이스, 도구, 지식 베이스 4가지 페이지를 포함한다.

---

## 1. 에이전트 (`/dashboard/{orgId}/agents`)

에이전트 목록과 상세 페이지. 왼쪽에 폴더 트리, 오른쪽에 선택한 에이전트의 상세.

### 1.1 목록/폴더 관리

- **폴더 생성**: 왼쪽 트리 상단 "+" 또는 우클릭 컨텍스트 메뉴
- **드래그 앤 드롭**: 에이전트를 폴더 간 이동, 폴더 계층 재정렬
- **검색**: 이름으로 빠르게 찾기

### 1.2 에이전트 생성

- "+ 새 에이전트" 버튼 → `/dashboard/{orgId}/agents/new`로 이동
- URL 쿼리로 폴더/타입 사전 선택 가능: `?folderId={id}&type=single_prompt`
- 타입: **single_prompt** (단일 프롬프트, 기본) | **flow** (노드 기반 흐름)

### 1.3 에이전트 상세 페이지 (3영역 구조)

에이전트 목록에서 에이전트를 클릭하면 상세 페이지가 열린다:

- **상단 바**: LLM 모델 선택, TTS 음성 선택, 언어 선택
- **왼쪽**: 프롬프트 편집기 (시스템 프롬프트 전문)
- **오른쪽 패널 (아코디언)**: 도구, 지식 베이스, 통화 정보 추출, 발화 설정, 통화 설정, 보안 설정, 웹훅 설정

#### 프롬프트 수정
- Markdown 형식 (`# 역할`, `# 컨텍스트`, `# Variables`, `# 목표`, `# 말투/발화 스타일` 등)
- `{{변수명}}` 런타임 변수 → 보라색 하이라이트
- 하단 **첫 메시지**: "AI가 먼저 말함(동적)" 또는 고정 메시지 선택
- 수정 후 상단 우측 **저장** 필수. 저장 전 페이지 이탈 시 변경사항 소실

#### 목소리(TTS)/속도
- 상단 바 음성 드롭다운 → 사용 가능한 음색 목록
- 음성 옆 톱니바퀴 아이콘 → 상세 TTS 설정
- **오른쪽 패널 > 발화 설정**:
  - 반응속도 슬라이더 (기본 1.0, 낮추면 응답 지연 증가)
  - 말 끼어들기 허용 토글
  - 키워드 강조 리스트 (에이전트가 특정 단어를 더 잘 인식)

#### 도구 설정
- 오른쪽 패널 > **도구** 섹션
- "+ 추가" → 빌트인(end_call, transfer_call, send_sms, send_dtmf 등) 또는 커스텀 도구 선택
- 각 도구 카드의 휴지통 아이콘으로 해제
- 도구 description으로 에이전트가 호출 시점 판단

#### 통화정보 추출
- 오른쪽 패널 > **통화 정보 추출** 섹션
- 통화 종료 후 자동 추출할 필드 정의
- "+ 추가" → 필드명 + 추출 기준(description) 입력
- 추출 결과는 **통화 기록** 테이블에 칼럼으로 자동 표시 (monitor.md 참고)

#### 테스트 패널 (오른쪽 상단)
- 에이전트 상세에서 **테스트 패널** 열기
- "음성" 탭 → **연결** 버튼으로 브라우저에서 바로 음성 대화
- 테스트는 크레딧 소모, 실제 발신 없이 즉시 검증
- **( ) 아이콘** → 테스트 프리셋 모달 (변수값 미리 채우기)
  - 시스템 변수(`agent_id`, `call_from`, `call_id`, `call_to`, `current_time`) 자동
  - 에이전트 변수(`{{고객명}}`, `{{브랜드명}}` 등) 수동 입력
  - 저장 후 연결 → 입력한 변수값이 적용된 상태로 대화 시작

#### 버전 배포
- 상단 우측 **배포** 버튼 → 에이전트 배포 기록 모달
- 하단 "설정" 버튼으로 현재 draft를 프로덕션 버전으로 배포
- 대량 발신 시 특정 버전 선택 가능 (current 또는 배포된 스냅샷)

## 2. 보이스 (`/dashboard/{orgId}/voice`)

보이스 라이브러리. 제공자(ElevenLabs, OpenAI 등)의 음색을 탐색하고, 자체 음성을 클론한다.

- **필터**: 제공자, 성별, 언어
- **재생/일시정지**: 카드의 플레이 버튼으로 샘플 청취
- **상세 프로필**: 이름, 설명, 샘플 URL
- **보이스 클론**: 딥링크 `?clone=true`로 클론 대화 자동 오픈
  - 오디오 파일 업로드 → 음성 합성 → 개인 라이브러리에 추가
- **수정/삭제**: 개인 보이스에 한함 (제공자 음성은 읽기 전용)

> 보이스 클론/편집/삭제는 UI에서만 가능. MCP 도구로는 제공자 목록 조회 정도.

## 3. 도구 (`/dashboard/{orgId}/tools`)

커스텀 도구 관리. 빌트인 도구(end_call 등)는 에이전트 상세에서 장착만 하면 되고, 커스텀 도구는 여기에서 정의한다.

### 3.1 목록 페이지
- 등록된 커스텀 도구 카드 목록
- 카드 클릭 → 상세 페이지 `/dashboard/{orgId}/tools/{toolUid}`

### 3.2 커스텀 도구 생성
- "+ 새 도구" 또는 딥링크로 폼 자동 오픈:
  - **API 도구**: `?create=api`
  - **MCP 도구**: `?create=mcp`

#### API 도구 폼 항목
- 이름, 설명
- Method(GET/POST/…), URL
- Headers, Query Params, Body
- 파라미터 스키마 (에이전트가 채울 값의 타입/설명)
- 인증 (없음/Bearer/Custom header)

#### MCP 도구 폼 항목
- 이름, 설명
- MCP 서버 URL
- 인증 (API key 등)
- 노출할 MCP tool 이름

> 도구 개념 설명, 이름 규칙, 빌트인 vs 커스텀 선택 기준 등은 `vox-tools` 스킬 참조.

## 4. 지식 베이스 (`/dashboard/{orgId}/knowledge`)

에이전트가 참조할 문서를 모아두는 공간. 에이전트 상세의 **오른쪽 패널 > 지식 베이스**에서 연결한다.

### 4.1 목록
- 지식베이스 카드 목록
- 검색/필터
- 딥링크 `?knowledgeId={id}`로 특정 KB 선택 상태로 열기

### 4.2 지식 베이스 상세 (`/dashboard/{orgId}/knowledge/{knowledgeUid}`)
- 문서 업로드 (드래그 앤 드롭 또는 파일 선택)
- 업로드된 문서 목록
- 텍스트 추출 결과 확인
- 문서 삭제
- 개별 문서 클릭 → 추출된 텍스트 미리보기

> CSV/PDF/DOCX 등 파일 업로드는 UI에서만 가능. MCP 도구로는 제공되지 않음.

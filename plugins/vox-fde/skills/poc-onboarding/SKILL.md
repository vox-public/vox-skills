# POC Onboarding Guide

vox.ai FDE가 고객 POC를 처음부터 끝까지 독립 수행할 수 있도록 돕는 스킬이다.

## 레퍼런스 역할 분담

이 스킬은 라우터다. 실제 내용은 아래 레퍼런스에 있으므로, 작업 전에 해당 파일을 반드시 읽는다.

| 파일 | 역할 | 언제 읽나 |
|------|------|-----------|
| `references/poc-milestones.md` | Linear 구조 생성 스펙 (모드 A/B, 이슈 목록, 생성 순서, 일정 산정) | POC 프로젝트/이슈를 **생성**할 때 |
| `references/poc-checklist.md` | 단계별 실행 가이드 (체크리스트, 노하우, 데모 전달물 템플릿, 공통 규칙) | 각 단계를 **실행**할 때 |

## 핵심 원칙

- FDE 업무 범위는 계약 이전의 POC 수행이다. 계약 관련 업무(계약서, 착수금, 잔금 등)는 범위 밖이다.
- 에이전트 프롬프트 작성 → `vox-best-practice` 스킬에 위임한다.
- 대화 흐름 노드 작성 → `flow-node-creator` 스킬에 위임한다.
- Linear 프로젝트/이슈 조작 → Linear MCP 도구를 사용한다.
- vox 에이전트 생성/수정 → vox MCP 도구를 사용한다.

## POC 단계 요약

| 단계 | 이름 | 생성 시점 |
|------|------|-----------|
| 1 | 초기 온보딩 | 초기 생성 |
| 2 | 에이전트 설계 및 제작 | 초기 생성 |
| 3 | 1차 데모 전달 | 초기 생성 |
| 4 | 피드백 & 수정 | 초기 생성 |
| 5 | 고객 온보딩 | 4단계 완료 후 |
| 6 | PoC 모니터링 | 4단계 완료 후 |
| 7 | PoC 종료 | 4단계 완료 후 |

## 사용 시나리오별 안내

### 새 POC 시작
1. `references/poc-milestones.md`를 읽고 Linear에 구조를 생성한다.
2. `references/poc-checklist.md`에서 1단계 가이드를 안내한다.

### 현재 단계 가이드 요청
1. Linear에서 해당 고객 프로젝트/이슈 상태를 조회한다.
2. `references/poc-checklist.md`에서 해당 단계의 가이드를 제공한다.

### 에이전트 작업이 필요할 때
- **프롬프트 작성/수정** → `vox-best-practice` 스킬의 워크플로우를 따른다.
- **대화 흐름 노드 작성** → `flow-node-creator` 스킬을 사용한다.

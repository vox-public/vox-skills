# 파트 2: 테스트하는 방법

에이전트를 실제 전화 없이 빠르게 검증하는 방법을 안내한다.

## 2.1 웹 테스트

에이전트 상세 페이지 오른쪽 상단의 **테스트 패널**을 사용한다.

- "음성" / "테스트" 두 탭이 있다. **음성** 탭에서 실제 음성 대화를 테스트할 수 있다.
- 하단의 **"연결"** 버튼을 클릭하면 브라우저에서 바로 에이전트와 음성 대화가 시작된다.
- 통화가 끝나면 테스트 패널에서 대화 로그를 확인할 수 있다.
- 웹 테스트는 크레딧을 소모하지만, 실제 전화를 걸 필요 없이 즉시 테스트할 수 있어 프롬프트 수정 후 빠른 검증에 유용하다.

## 2.2 {{변수}} 프리셋 설정 후 테스트

테스트 패널 오른쪽 상단의 **( ) 아이콘**을 클릭하면 **테스트 프리셋** 모달이 열린다.

**프리셋이란:** 에이전트 프롬프트에 `{{변수명}}` 형태로 정의된 변수에 테스트용 값을 미리 채워넣는 기능이다.

**변수 종류:**

- **시스템 변수** (자동 대체, 수정 불필요): `agent_id`, `call_from`(발신 전화번호), `call_id`(통화 ID), `call_to`(수신 전화번호), `current_time`(현재 날짜/시간)
- **에이전트 변수** (직접 값 설정): 프롬프트에서 `{{변수명}}`으로 정의한 변수들. 예: `고객명` = "김민철", `브랜드명` = "복스에이아이"

**사용 방법:**

1. ( ) 아이콘 클릭 → 테스트 프리셋 모달 열기
2. 에이전트 변수 값 입력 (시스템 변수는 자동이므로 건드리지 않아도 됨)
3. **저장** 클릭
4. 테스트 패널에서 **연결** → 입력한 변수값이 적용된 상태로 대화 시작

프리셋을 설정하면 "고객님" 대신 "김민철님"처럼 실제 발신 상황과 동일한 조건으로 테스트할 수 있다.
